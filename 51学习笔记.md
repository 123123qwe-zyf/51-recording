## 模式1
THx（高 8 位） + TLx（低 8 位），共 16 位，计数范围 0~65535
*必须手动重装初值*
工作过程
1. 给 THx、TLx 设置初值；
2. 启动定时器后，TLx 从初值开始加 1，TLx 溢出则 THx 加 1；
3. THx、TLx 都溢出（计满 65536 次）→ 触发定时器中断；
4. 中断后需**手动重装初值**（否则下次计数从 0 开始，定时不准）
**TCON（定时器控制寄存器，地址 88H，可位寻址）** 
==启动 T0： TR0 = 1      停止  TR0 = 0==
**IE（中断允许寄存器，地址 A8H，可位寻址 
- 常用配置（开启 `T0` 中断）：==EA = 1; ET0 = 1;==
晶振11.0592  `1ms` 定时中断
```
void Timer0_Init(void)
{
	TMOD &=0xF0;    //清空T0模式位
	TMOD |=0x01;      //T0模式1（16位定时
	TH0=0xFC;          //初值高8位
	TL0=0x67;          //初值低8位
	ET0=1;             //开启T0中断
	EA=1;              //开启总中断
	TR0=1;             //启动T0
}
``` 

# 通信
**并行通信**
多条数据线同时传输多位数据  每一位占用一条独立的物理线路  
**串行通信**
数据按位顺序，在一条或两条线路上逐位传输  UART、SPI、I²C、USB、以太网等

**异步通信**（`UART`）
收发双方不共享统一时钟，通过起始位、停止位等帧格式来同步
以**字符帧**为单位传输
**帧结构**：起始位（1 位，低电平）+ 数据位（5-8 位）+ 校验位（可选）+ 停止位（1-2 位，高电平）
双方必须预先约定==相同==的波特率
**同步通信**（SPI、I²C、SD 卡、以太网）
收发双方共享同一时钟信号，数据在时钟节拍下连续传输
以**数据块**为单位传输，时钟信号由主机提供或双方共享
**`UART`硬件结构**
**`TXD（P3.1）`**：串行数据发送引脚
**`RXD（P3.0）`**：串行数据接收引脚
**`SBUF`（串行数据缓冲器）**：两个物理上独立的寄存器，一个用于发送，一个用于接收，共用地址` 0x99`
**`SCON`（串行控制寄存器）**：控制 `UART` 的工作方式、帧格式和状态标志
**定时器 / 计数器 1**：通常作为波特率发生器，为` UART` 提供时钟
帧格式：1 位起始位 + 8 位数据位 + 1 位停止位，无校验位
==寄存器配置==
`SCON = 0x50`(收发双向)  0x40只能发送
![[Pasted image 20260225232045.png]]
`REN `接收允许位 `REN = 1`：允许从` RXD `引脚接收数据  0禁止
`PCON`  `0x87`，不可位寻址    `PCON &= 0x7F` → 清零最高位
`TMOD`  `0x89`，不可位寻址 
关键位 `SMOD`  等于0波特率不加倍 -
`TMOD &= 0x0F` → 清空` T1` 的模式位（高 4 位）
`TMOD |= 0x20` → 设置 `T1` 为模式 2（8 位自动重装），这样定时器溢出后自动从 TH1 重装初值，保证波特率稳定
![[Pasted image 20260225232150.png]]

**单工通信**   数据只能在一个方向上传输
**半双工通信**  数据可以双向传输，但同一时刻只能在一个方向上传输，收发不能同时进行
**全双工通信**   数据可以同时在两个方向上传输，收发双方可以同时发送和接收数据

```c
void UART_SendByte(unsigned char dat)
{
    SBUF = dat;         // 写入数据，启动发送
    while(TI == 0);     // 等待发送完成
    TI = 0;             // 必须软件清0！
}
// 接收一个字符（查询方式，阻塞）
unsigned char UART_ReceiveByte(void)
{
    while(RI == 0);     // 等待接收完成
    RI = 0;             // 必须软件清0！
    return SBUF;
}
```
